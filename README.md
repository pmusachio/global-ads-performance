## Business Context

Digital advertising in e-commerce operates under two fundamental tensions: **which campaign format
drives the highest return**, and **how to distribute a fixed budget across channels to maximise
that return**.

This project addresses both questions on a dataset of daily ad performance records across three
platforms (Google Ads, Meta Ads, TikTok Ads), three campaign types (Search, Shopping, Video)
and multiple industries — with the e-commerce vertical as the primary focus.

**Variants under test**

| Group | Definition |
|-------|------------|
| Group A | Search campaigns — intent-driven, lower funnel |
| Group B | Video campaigns — awareness-driven, upper funnel |

**Primary metrics**

| Metric | Description |
|--------|-------------|
| ROAS | Return on Ad Spend = Revenue / Ad Spend |
| Conversion Rate | Conversions / Clicks |
| Average Ticket | Revenue / Conversions |

## Methodology

### A/B Test Design

The sample size required to detect a real effect is calculated before the test, using Cohen's
formula for a two-sample test:

```
n = 2 * ((z_alpha/2 + z_beta) / d)^2
```

where `d` is Cohen's standardised effect size, `z_alpha/2` corresponds to a significance level
of 5% (two-sided) and `z_beta` corresponds to 80% statistical power.

### Assumption Validation

| Assumption | Test | Decision rule |
|------------|------|---------------|
| Normality | Shapiro-Wilk (n <= 5 000) or D'Agostino-Pearson (n > 5 000) | p > 0.05 -> normal |
| Variance homogeneity | Levene's test (only if both groups are normal) | p > 0.05 -> equal variances |

### Test Selection

The statistical test is chosen automatically based on the validation outcomes:

| Condition | Test applied |
|-----------|-------------|
| Both groups normal, equal variance | Student's t-test |
| Both groups normal, unequal variance | Welch's t-test |
| At least one group non-normal | Mann-Whitney U (non-parametric) |

Effect size is reported as Cohen's d (parametric) or rank-biserial correlation (non-parametric),
and a 95% confidence interval is computed for the difference in means.

### Channel Revenue Modeling

Each platform is modelled with a two-stage scikit-learn pipeline:

1. **AdstockTransformer** — geometric decay of ad carryover:
   `effective_spend[t] = spend[t] + decay_rate * effective_spend[t-1]`

2. **SaturationRegressor** — diminishing-returns curve fitted with `scipy.optimize.curve_fit`:
   `revenue = L * (1 - exp(-k * spend))`

Parameters are estimated on a chronological 70% training split and evaluated on the held-out
30% test window to prevent look-ahead bias.

### Budget Optimization

Given the fitted saturation parameters `(L, k)` per channel, the optimal daily allocation is
found by solving a constrained optimisation problem with SLSQP
(`scipy.optimize.minimize`):

- **Maximise** total predicted revenue
- **Subject to:** total spend <= budget cap; global ROAS >= floor; each channel between
  `min_share` and `max_share` of total budget

## How to Run

The entire project runs on **Google Colab** — no local Python installation required.

1. Click the badge at the top of this README:

   [![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/pmusachio/global-ads-performance/blob/main/global-ads-performance.ipynb)

2. In Colab, go to **Runtime > Run all** (Ctrl+F9).

   The first cell automatically:
   - Clones the repository to `/content/global-ads-performance/`
   - Downloads the dataset from Google Drive
   - Adds the repository root to `sys.path` so all auxiliary modules are importable

3. All subsequent cells execute sequentially and produce the full analysis, including plots
   saved to `assets/`.
